2. 표면적 수준에서의 코드 개선
* 보편적 이름 피하기
  - tmp : temporary
  - get : fetch, download
  - size : height, numNodes, memoryBytes 

* 루프 반복자 이름 짓기
  - i, j, k : club_i(ci), member_i(mi), user_i(ui)

* 불린 변수 이름 짓기
  - is, has, can, should
  - positive expressions needed

* 범위값을 나타내는 변수명 짓기
  - 양단 포함 : min/max, first/last, left/right
  - 뒷단 불포함 : begin/end

3. 로직의 개선
* 조건문에서 인수의 순서
  - 유동적인/비교가 되어지는, 고정적인/비교대상으로 사용되는
  - 변수, 수
  - received, expected

* if/else 블록의 순서
  - 긍정이 먼저
  - 간단한 것 먼저
  - 더 중요한 것 먼저

* 블록의 중첩 최소화
  - if (1) { if (1) { ... }} x
